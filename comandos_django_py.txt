PASSO A PASSO DO PROJETO DJANGO
USANDO SQLITE LOCAL E DADOS COMPARTILHADOS EM JSON

1. PRIMEIRO USO NA MAQUINA

# Criar ambiente virtual
py -m venv venv

# Ativar ambiente virtual no Windows
venv\Scripts\activate

# Atualizar pip
py -m pip install --upgrade pip

# Instalar Django
py -m pip install django

# Se um dia o projeto tiver requirements.txt, usar:
# py -m pip install -r requirements.txt


2. SEMPRE QUE ABRIR O PROJETO

# Ativar ambiente virtual
venv\Scripts\activate


3. CRIAR OU ATUALIZAR O BANCO LOCAL

# Criar migrations a partir dos models
py manage.py makemigrations

# Aplicar migrations e montar o banco na sua maquina
py manage.py migrate


4. RODAR O PROJETO

py manage.py runserver


5. PASTA DE BACKUP

# A pasta backups ja existe no projeto
# Os arquivos de backup devem ficar dentro dela


6. EXPORTAR OS DADOS PARA O GRUPO

# Recomendado: exportar so as perguntas do quiz
py manage.py dumpdata quiz.Pergunta --indent 2 > backups\perguntas.json

# Se quiser exportar o banco inteiro
py manage.py dumpdata --indent 2 > backups\dados.json


7. IMPORTAR OS DADOS EM OUTRA MAQUINA

# Primeiro montar a estrutura do banco
py manage.py migrate

# Depois carregar as perguntas
py manage.py loaddata backups\perguntas.json

# Se o grupo estiver usando o banco inteiro em JSON
py manage.py loaddata backups\dados.json


8. FLUXO CERTO PARA O GRUPO

# Quem atualizou as perguntas:
# 1. faz o dump para JSON
# 2. sobe o arquivo JSON para o GitHub

# Quem recebeu do GitHub:
# 1. faz pull
# 2. roda migrate
# 3. roda loaddata


9. O QUE VAI PARA O GITHUB

# Vai para o GitHub:
# - codigo
# - migrations
# - requirements.txt
# - backups\perguntas.json
# - ou backups\dados.json

# Nao vai para o GitHub:
# - db.sqlite3
# - venv
# - __pycache__


10. RESUMO BEM CURTO

# Cada pessoa cria o proprio banco com:
py manage.py migrate

# E coloca os mesmos dados com:
py manage.py loaddata backups\perguntas.json

# Assim:
# db.sqlite3 fica local
# perguntas.json vai para o GitHub
